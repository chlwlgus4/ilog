# 아이로그 요청 템플릿

Codex에 요청할 때는 아래 두 형식 중 작업 성격에 맞는 템플릿을 사용합니다.

- 새 화면, 기능 추가, 데이터 구조 변경: `기능 요청`
- 기존 동작의 오류, 화면 깨짐, 사용성 개선: `수정 요청`

`[]` 안의 안내 문구는 지우고 실제 내용으로 바꿔서 전달합니다. 모르는 항목은 비워도 되며, 그 경우 필요한 확인 사항만 질문합니다.

---

## 기능 요청

```md
[기능] [기능 이름]
예: 가족 사진 앨범 업로드 개선

## 목표
- [사용자가 얻어야 하는 결과]

## 대상 화면 / 경로
- [예: 더보기 > 사진 앨범]
- [예: 가족 대화]

## 원하는 동작
1. [사용자 행동과 결과]
2. [사용자 행동과 결과]
3. [사용자 행동과 결과]

## 데이터 / 권한
- 조회 가능 대상: [예: 같은 가족 구성원 전체]
- 생성 가능 대상: [예: 로그인한 가족 구성원]
- 수정 / 삭제 가능 대상: [예: 업로더 본인만]
- 기존 데이터 처리: [예: 기록 첨부 사진은 읽기 전용으로 유지]

## 예외 / 제한
- [예: 한 번에 최대 10장 업로드]
- [예: 업로드 실패 항목은 성공 항목과 구분해 표시]
- [예: 삭제 전 확인 모달 표시]

## 건드리지 말 것
- [예: 기존 기록 저장 흐름은 유지]

## 완료 기준
- [예: 여러 장 선택, 업로드, 확대 보기, 삭제가 모두 동작]
- [예: 권한 없는 사용자는 삭제할 수 없음]

## 검증
- 실제 테스트: [예: 업로더와 다른 가족 계정으로 각각 확인]
- 화면 확인: [예: 모바일 웹 또는 iOS / Android]

## 배포
- Supabase 변경: [없음 / migration, function, storage policy 배포 필요]
- 앱 빌드: [없음 / preview 또는 production 빌드 필요]
- 커밋 / 푸시: [필요 / 불필요]
```

## 수정 요청

```md
[수정] [문제 이름]
예: 통계 화면 차트 하단 잘림

## 문제
- [현재 발생하는 현상]

## 기대 결과
- [수정 후 보여야 하거나 동작해야 하는 결과]

## 재현 방법
1. [예: 통계 탭으로 이동]
2. [예: 수면 또는 수유 카테고리 선택]
3. [예: X축 날짜 라벨 하단이 잘리는지 확인]

## 대상 화면 / 경로
- [예: 하단 탭 > 통계]

## 참고 자료
- 첨부 화면: [있음 / 없음]
- 유지할 문구 또는 디자인: [예: 기존 색상과 차트 데이터는 유지]

## 건드리지 말 것
- [예: 통계 계산 방식과 기간 선택 동작은 유지]

## 완료 기준
- [예: iPhone 화면 너비에서 모든 X축 라벨이 잘리지 않음]
- [예: 다른 차트 종류에도 같은 문제가 없음]

## 검증 / 배포
- 실제 테스트: [필요 / 불필요]
- Playwright 화면 확인: [필요 / 불필요]
- Supabase 변경: [없음 / 필요]
- 커밋 / 푸시: [필요 / 불필요]
```

## 한 번에 여러 요청할 때

서로 독립적인 작업이면 한 메시지 안에서도 요청을 분리합니다. 우선순위와 커밋 범위를 함께 적으면 작업 순서가 더 명확해집니다.

```md
[요청 1: 기능] 가족 사진 앨범 업로드 개선
[요청 2: 수정] 통계 화면 차트 하단 잘림

## 우선순위
1. 요청 2
2. 요청 1

## 커밋 범위
- 두 요청을 하나의 커밋으로 처리
```

요청 1의 결과를 요청 2에서 사용해야 한다면, `요청 1 완료 후 요청 2 진행`이라고 적습니다.
